When would you want to use a remote repository rather than keeping all your work
local?

    You would use a remote repository when you were sharing your work with others

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    If your local repo represents a production application and you can't allow changes to take place without testing

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

    A fork is a copy of a repo from one place on github to another.  Use a fork when you need your own copy of a repo.  A clone is a copy of a repo when initializing the repo on your local machine.  A branch is a named commit.
